# 提案：精简后端为 Codex/Pi 并迁移测试

## 背景

工作流 CLI 的核心价值是稳定执行 oz change。保留已经不用的第三方后端会让配置校验、runner 注册、状态展示、测试夹具和规格文档都变复杂。用户已经明确不需要历史兼容，目标是最大程度精简和高效。

测试布局也需要同步收敛。`internal/` 应只表达生产源码；长期测试集中到根目录 `docs/changes/archive/2026-06-11-14-精简后端为-codex-pi-并迁移测试/tests/`，便于发布门禁、规格测试归档和执行器发现。

## 变更内容

- agent backend allowlist 收窄为 `codex` 和 `pi`。
- 删除旧后端所有实现、测试、集成入口、状态兼容和文档描述。
- `tool`/`cli` 配置只接受 `codex` 和 `pi`，其他值按未知工具失败。
- sealed run 启动前统一检查 `codex` 和 `pi` 两个 CLI 是否存在；缺失时输出清晰安装提示，并且不创建 `runs/` 状态。
- `internal/**/_test.go` 全部迁移到根目录 `docs/changes/archive/2026-06-11-14-精简后端为-codex-pi-并迁移测试/tests/` 下按业务能力组织。
- 更新主规格和发布测试，保证后端集合和测试布局不会回退。

## 不做

- 不保留旧配置兼容。
- 不保留旧状态兼容。
- 不保留旧 session key 展示。
- 不保留第三后端专属测试作为回归保护。

## 验收方式

本提案以 shell 契约测试为硬合同：

- 仓库当前内容不再包含旧后端字样或实现文件。
- CLI 启动前必须同时检查 `codex` 和 `pi`。
- `internal/` 下不存在长期测试文件。
- 文档和发布门禁只承诺 Codex/Pi 以及根目录测试。
