# 文件目的

本文件记录收敛迁移测试合同的技术方案、取舍和风险。

## 技术方案

1. 以 `go test ./...` 当前失败为入口，列出失败 `.gotest` 断言。
2. 对每个失败断言判断归属：
   - 当前业务仍需要：迁入真实测试包并修正生产代码。
   - 当前业务已经改变：按新规格改写测试或删除历史断言。
3. 保留必要测试辅助函数，但移到真实测试文件或共享 test helper。
4. 移除 `docs/changes/archive/2026-06-14-20-收敛迁移测试合同/tests/app/migrated_app_suite_test.go` 的临时复制执行路径，或让它在没有 `.gotest` 时不再承担门禁。

## 取舍

不继续维护 `.gotest` 迁移层。它降低短期迁移成本，但现在已经变成不稳定信号源。

## 风险

- 迁移时可能误删仍有效的回归测试。缓解方式是每个删除动作都必须能在提交说明或测试命名中说明对应旧意图已由哪个真实测试覆盖。
- 根测试耗时可能短期增加。缓解方式是优先复用现有 Go 测试包，不新增外部依赖。
