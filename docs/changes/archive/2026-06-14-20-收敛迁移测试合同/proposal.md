# 文件目的

本文件说明为什么要先收敛迁移测试合同，以及本次变更要交付什么。

## 背景

仓库正在进入稳定性优先的重构阶段。当前根测试门禁包含一层 `.gotest` 迁移包装，它把 `tests/app/*.gotest` 临时复制回 `internal/app` 执行。这个机制曾用于迁移测试，但现在已经出现与当前实现意图不一致的断言。

## 问题

当旧迁移合同失败时，开发者无法判断失败来源：

- 真实业务行为回归。
- 历史测试没有跟随新意图更新。
- 临时复制环境与真实包测试环境不一致。

这会直接降低后续重构安全性。

## 变更

- 逐个审计 `tests/app/*.gotest`。
- 有效测试迁入真实 Go 测试文件。
- 过期测试按当前规格改写或删除。
- 根目录 `go test ./...` 不再依赖 `.gotest` 临时迁移机制。

## 稳定性原则

本提案不得通过跳过根测试、缩小 CI 范围、删除有效断言或只改测试期望来制造通过结果。每个保留测试都必须对应当前真实业务合同。
